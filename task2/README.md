### 1. Nine Pillars Understanding
**Q1A. Why is using AI Development Agents (like Gemini CLI) for repetitive setup tasks
better for your growth as a system architect?**  
AI Development Agents  repetitive setup aur boilerplate ka kaam karte hain. Is se hum developers zyada focus architecture, design aur problem-solving pe kar sakte hain.

**Q1B.Explain how the Nine Pillars of AIDD help a developer grow into an M-Shaped Developer.**  
M-Shaped Developer aesa developer jo ek se zyada related domains me deep expertise rakhta ho aur AI tools ke sath efficiently complex systems develop kar sakta ho.Nine Pillars (Specification-Driven, AI-Augmented, Agent-Orchestrated, Quality-Gated, Version-Controlled, Human-Verified, Iteratively-Refined, Documentation-Embedded, Production-Ready)
ek complete system banate hain. Ye hume multiple skills develop karne aur solo me complex systems handle karne me help karte hain  is tarah M-Shaped Developer banta hai.

---

### 2. Vibe Coding vs Specification-Driven Development
**Q2A. Why does Vibe Coding usually create problems after one week?**  
Vibe Coding = bina plan ya spec ke code likhna. Is se code inconsistent, hard to maintain aur buggy ho jata hai, specially jab project bada ho jaye.

**Q2B. How would Specification-Driven Development prevent those problems?**  
SDD means pehle clear specifications likhna. Ye AI aur hum dono ko exact instructions deta hai, errors kam hote hain, structure consistent hota hai aur future maintenance easy ho jata hai.

---

### 3. Architecture Thinking
**Q3A. How does architecture-first thinking change the role of a developer in AIDD?**  
Architecture-first thinking se developer ka role sirf coding tak limited nahi rehta. Is approach mein developers ka focus system design ki taraf shift ho jata hai. Kyunke AI code likh leta hai, isliye developers ko code implementation ki fikar nahi hoti. Unka main kaam overall system structure, layers, workflows, aur agent interactions ko design aur define karna hota hai.AI implementation handle karta hai, jabke developer high-level planning, architecture, aur system behavior ko control karta hai yani developer ki role coding se zyada solution architect jaisa ho jata hai

**Q3B. Explain why developers must think in layers and systems instead of raw code.**  
Developers ko raw code ki bajaye layers aur systems mein sochna isliye zaroori hai kyunke AIDD mein complete workflow layers pe operate karta hai.
3-Layer Architecture (Models ‚Üí IDE/Workspace ‚Üí Agents) developer ko clearly batati hai ke system ka har part kis responsibility ka hai.
Is approach se developer scalable, maintainable aur production-ready systems design kar sakta hai, sirf syntax ya choti scripts pe depend nahi karta.
